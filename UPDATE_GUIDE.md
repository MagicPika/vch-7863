# Как обновлять сайт В/Ч №7863

Все основные тексты и списки лежат в одном файле:

```txt
src/data/mockData.ts
```

## 1. Обновить уставы

Откройте `src/data/mockData.ts` и найдите блок:

```ts
export const STATUTES = [
```

Каждая глава выглядит так:

```ts
{
  id: 'orders',
  title: 'Глава 4. Приказ (Распоряжение)',
  description: 'Порядок отдачи, исполнения и обжалования приказов.',
  articles: [
    { num: '§17', text: 'Текст статьи...' },
    { num: '§18', text: 'Текст статьи...' }
  ]
}
```

Чтобы добавить статью, добавьте строку внутрь `articles`:

```ts
{ num: '§44', text: 'Новый текст статьи.' },
```

Чтобы добавить новую главу, скопируйте целый блок главы и поменяйте `id`, `title`, `description` и `articles`.

## 2. Обновить личный состав

В том же файле найдите:

```ts
export const INITIAL_SOLDIERS = [
```

Карточка человека выглядит так:

```ts
{
  id: 's21',
  name: 'Фамилия Имя Отчество',
  rankId: 8,
  division: 'ВП',
  position: 'Должность',
  discord: '@роль | Ник | ID',
  status: 'В СТРОЮ',
  warnings: 0,
  medals: [],
  joinDate: '2026-01-01',
  avatar: '/images/people/example.png'

```

`rankId` берется из блока `RANKS` выше.

`division` должен быть одним из значений:

```txt
Штаб В/Ч, ВП, 9-я рота, 12-й бат., Военкомат, МТО, Академия
```

## 3. Обновить подразделения

Найдите блок:

```ts
export const DIVISIONS = {
```

Подразделение выглядит так:

```ts
'ВП': {
  name: 'Военная полиция',
  desc: 'Описание подразделения...',
  image: '/images/divisions/military-police.svg'
}
```

Чтобы поменять описание, редактируйте `desc`.

Чтобы поменять картинку, редактируйте `image`.

## 4. Заменить эмблемы подразделений

Эмблемы лежат здесь:

```txt
src/assets/divisions/
```

Файлы:

```txt
headquarters.svg     - Штаб В/Ч
military-police.svg  - ВП
airborne-special.svg - 9-я рота СПН ВДВ
motorized.svg        - 12-й батальон
commissariat.svg     - Военкомат
logistics.svg        - 101-й МТО
academy.svg          - Военная академия
```

### Самый простой способ — заменить файл

Просто положите свою эмблему в `src/assets/divisions/` с тем же именем (например, `military-police.svg`) — она автоматически заменится.

Поддерживаются форматы:

```txt
.png, .jpg, .jpeg, .svg, .webp
```

Лучше всего работают вертикальные эмблемы с прозрачным фоном (PNG или SVG).

### Если хотите свои имена файлов

1. Положите файл, например:
```txt
src/assets/divisions/my-emblem.png
```

2. Откройте `src/data/mockData.ts` и в самом верху добавьте импорт:
```ts
import myEmblem from '../assets/divisions/my-emblem.png';
```

3. В нужном подразделении в `DIVISIONS` пропишите:
```ts
image: myEmblem
```

⚠️ Важно: эмблемы импортируются как модули, а не лежат в `public/`. Это нужно, чтобы они работали на GitHub Pages с подпапкой.

## 8. Пароль для входа в ЕИС МО (закрытый портал)

Сразу под ссылкой портала в `src/App.tsx`:

```ts
const STAFF_PASSWORD = 'ВЧ7863';
```

### 🕵️ Как зайти (секретные способы)

ЕИС спрятан — посторонние НЕ видят кнопку входа. Свои входят так:

**Способ 1 — 5 кликов на герб**
Нажмите 5 раз подряд на герб В/Ч в шапке сайта (за 2 секунды). Откроется окно ввода пароля.

**Способ 2 — горячая клавиша**
Нажмите `Ctrl + Shift + L` на любой странице сайта. Сразу откроется окно ввода пароля.

После ввода правильного пароля:
- Браузер запоминает авторизацию
- В шапке появляется зелёная кнопка **«Открыть портал»**
- На главной появляется большая зелёная карточка **«ЕИС МО РФ»**
- Чтобы выйти — нажать «Выйти» в шапке

### 📢 Что говорить новым военнослужащим

Когда принимаете нового бойца, скажите ему:

> «Чтобы попасть в ЕИС, на главной странице сайта нажми 5 раз на герб (или Ctrl+Shift+L). Введи пароль `ВЧ7863`. Дальше всё будет открываться автоматически.»

Замените `ВЧ7863` на любой пароль, который вы хотите выдавать военнослужащим.

### Как это работает:

- На сайте есть кнопка **«Вход для л/с»** (золотая, в шапке)
- При нажатии открывается окно ввода пароля
- Если пароль правильный — открывается портал в новой вкладке
- Браузер запоминает авторизацию, и в следующий раз пароль вводить не нужно (пока пользователь не нажмёт «Выйти»)
- Без пароля сам адрес портала недоступен — посторонние его не увидят

### Важно про безопасность:

Пароль хранится в коде сайта и теоретически продвинутый пользователь может его подсмотреть через DevTools. Это **защита от случайных посетителей**, не от хакеров. 

Для серьёзной защиты сам внутренний портал должен иметь свою авторизацию (логин/пароль на стороне сервера). Эта кнопка — просто фильтр, чтобы посторонние не лезли.

### Когда менять пароль:

- Если пароль попал к посторонним
- При смене состава командования
- Раз в 1-2 месяца для профилактики

После смены пароля старая авторизация у пользователей сохранится (они уже залогинены). Чтобы у всех слетела авторизация и пришлось вводить новый пароль — нужно поменять имя ключа в localStorage. Скажите мне «сбрось авторизацию у всех» — я подскажу как.
