# Как залить сайт на GitHub Pages

Самый простой способ — GitHub сам будет собирать сайт, вам нужно будет только заливать новые правки.

## 🚀 Первая публикация (один раз)

### Шаг 1. Создать репозиторий на GitHub

1. Зайдите на https://github.com → нажмите **«+»** в правом верхнем углу → **New repository**
2. Название: например `vch-7863` (или любое другое)
3. Выберите **Public** (для GitHub Pages нужно)
4. **НЕ** ставьте галочки на README, .gitignore, license — оставьте пустым
5. Нажмите **Create repository**

### Шаг 2. Залить код

GitHub покажет вам команды. Выполните в терминале **в папке проекта**:

```bash
git init
git add .
git commit -m "Первая версия сайта В/Ч 7863"
git branch -M main
git remote add origin https://github.com/ВАШ_НИК/vch-7863.git
git push -u origin main
```

Замените `ВАШ_НИК` на ваш ник GitHub.

### Шаг 3. Включить GitHub Pages

1. Откройте репозиторий на GitHub
2. **Settings** (в верхнем меню) → **Pages** (в левом меню)
3. В разделе **Build and deployment** → **Source**: выберите **GitHub Actions**
4. Готово, ничего больше выбирать не надо

### Шаг 4. Подождать сборку (~1 минута)

1. Перейдите во вкладку **Actions** в репозитории
2. Увидите запущенный процесс **Deploy to GitHub Pages**
3. Когда напротив появится зелёная галочка ✅ — сайт готов

### Шаг 5. Открыть сайт

Адрес будет:

```
https://ВАШ_НИК.github.io/vch-7863/
```

## 🔄 Обновление сайта (каждый раз когда что-то меняете)

После любых правок в файлах просто выполните в терминале:

```bash
git add .
git commit -m "Что обновили"
git push
```

GitHub автоматически пересоберёт сайт за ~1 минуту.

Например:
```bash
git add .
git commit -m "Обновил командование ВП"
git push
```

## 📝 Что и где менять

Все основные данные — в одном файле:

```
src/data/mockData.ts
```

Подробная инструкция в файле `UPDATE_GUIDE.md`.

## 🎨 Картинки подразделений

Кладите в папку:

```
public/images/divisions/
```

Поддерживаются `.png`, `.jpg`, `.svg`, `.webp`.

## 🌐 Свой домен (опционально)

Если у вас есть свой домен (например `armiya7863.ru`):

1. **Settings** → **Pages** → **Custom domain**
2. Введите домен и сохраните
3. У регистратора домена пропишите CNAME-запись на `ВАШ_НИК.github.io`

## ❓ Если что-то не работает

### Не пушится код:
Установите Git: https://git-scm.com/download/win

### Просит логин и пароль при push:
GitHub больше не принимает обычные пароли. Нужен **Personal Access Token**:
1. https://github.com/settings/tokens → **Generate new token (classic)**
2. Поставьте галочку **repo** → создать
3. Скопируйте токен
4. Используйте токен вместо пароля при `git push`

### Сайт не открывается:
- Проверьте во вкладке **Actions** что сборка прошла без ошибок
- Подождите 2-3 минуты после первой публикации
- Адрес должен быть с `/` на конце: `https://ВАШ_НИК.github.io/vch-7863/`

### Картинки не отображаются:
- Проверьте, что путь начинается с `/images/...`
- Файл должен быть в папке `public/images/...`
- Имена файлов чувствительны к регистру: `Vdv.png` ≠ `vdv.png`

## 💡 Удобные программы

Чтобы не работать в терминале:

- **GitHub Desktop** — https://desktop.github.com (визуально пушить изменения)
- **VS Code** — встроенная работа с Git, можно править код

С GitHub Desktop процесс ещё проще:
1. Сделали правки в файлах
2. Открыли GitHub Desktop
3. Видите все изменения
4. Внизу пишете комментарий → **Commit to main**
5. Сверху нажимаете **Push origin**
6. Через 1 минуту сайт обновлён
