# 🚀 Деплой на Vercel + Discord OAuth

## Шаг 1. Залить новые изменения на GitHub

В терминале в папке проекта:

```bash
git add .
git commit -m "Добавлен Discord OAuth"
git push
```

## Шаг 2. Импортировать проект в Vercel

1. Зайдите на https://vercel.com → войдите через GitHub
2. На главной нажмите **`Add New...`** → **`Project`**
3. В списке репозиториев найдите ваш репозиторий с сайтом → нажмите **`Import`**
4. На странице настроек:
   - **Framework Preset:** должен автоматически определиться как **Vite** ✅
   - **Build Command:** `npm run build` (по умолчанию)
   - **Output Directory:** `dist` (по умолчанию)
5. **НЕ нажимайте Deploy сразу!** Сначала добавьте секреты:

## Шаг 3. Добавить Environment Variables (секреты)

На той же странице нажмите **`Environment Variables`** и добавьте две переменные:

### Переменная 1
- **Name:** `DISCORD_CLIENT_ID`
- **Value:** `1501861655028432996`

### Переменная 2
- **Name:** `DISCORD_CLIENT_SECRET`
- **Value:** [ваш CLIENT SECRET — НОВЫЙ, после Reset]

⚠️ **Внимание:** старый CLIENT SECRET, который вы прислали в чат, нужно **сбросить** в Discord Developer Portal (Reset Secret) и получить новый — иначе он скомпрометирован.

Нажмите **`Add`** для каждой переменной.

## Шаг 4. Деплой

Нажмите большую кнопку **`Deploy`**.

Через 30-60 секунд получите адрес вида:
```
https://vch-7863-portal.vercel.app
```

## Шаг 5. Прописать Redirect URL в Discord

Скопируйте адрес сайта от Vercel (например `vch-7863-portal.vercel.app`) и:

1. Откройте https://discord.com/developers/applications → ваше приложение
2. **OAuth2** → **Redirects** → удалите старые, добавьте:
   ```
   https://vch-7863-portal.vercel.app/api/auth/callback
   ```
   (замените на ваш реальный адрес от Vercel)
3. Если хотите тестировать локально — оставьте также:
   ```
   http://localhost:5173/api/auth/callback
   ```
4. **Save Changes**

## Шаг 6. Проверка работы

1. Откройте ваш сайт от Vercel
2. Нажмите 5 раз на герб (или `Ctrl+Shift+L`)
3. Появится окно с кнопкой **«Войти через Discord»**
4. Нажмите → перебросит на Discord → согласитесь
5. Вернётесь на сайт → в шапке появится ваш аватар, имя, звание и подразделение
6. Нажмите **«ЕИС МО»** → откроется закрытый портал

---

## 🔄 Как обновлять сайт

Просто `git push` в репозиторий — Vercel **автоматически** пересоберёт сайт за 30 секунд.

```bash
git add .
git commit -m "Обновил состав"
git push
```

Vercel сам найдёт изменения и обновит сайт.

## ⚙️ Важно про DOMAIN

После того как Vercel даст адрес `vch-7863-portal.vercel.app`:

- Этот адрес работает сразу
- Можно **подключить свой домен** (например `vch7863.ru`) — бесплатно через Vercel → Settings → Domains
- GitHub Pages при этом продолжит работать — параллельно. Решите потом, какой адрес оставить главным.

## 🆘 Если что-то не работает

### «Войти через Discord» открывается, но возвращает ошибку
- Проверьте что Redirect URL в Discord **точно совпадает** с адресом Vercel (включая `https://`)
- Проверьте что в Vercel задан `DISCORD_CLIENT_SECRET`

### Аватар не отображается
- Discord может временно не отдавать аватары — попробуйте перелогиниться

### "User not in server"
- Это значит вы вошли, но не состоите в Discord-сервере с ID `809037474771763240`. Зайдите на сервер и попробуйте снова.
